Week 1
Jul 13–19
Phase 0 · Foundations

Foundations in one week: dev environment, data model, identity — the skeleton walks.

No user-facing features yet, but by Friday you can log in with CWL, the session persists, and a role-appropriate home renders. The scaffold everything else stands on.

Docker dev stack Pinned deps + typed config Shared domain types Collections + indexes PUID identity + login API contract + CI
🏁 Phase 0 exit — walking skeleton runs on both machines from a fresh clone.

WS-1Platform & auth — docker-compose (MongoDB, Qdrant, mock SAML IdP), pin ubc-genai-toolkit-* exact, typed env.ts + assertConfig(), helmet + rate-limit + zod, PUID-keyed login (session stores PUID only), role-appropriate home stub, lint + GitHub Actions CI.

Week 2
Jul 20–26
Phase 1 · Core loop

The whole loop, end to end: students practice with adaptive feedback; instructors generate & approve.

The irreducible core the pilot depends on — everything after it is additive. Students enroll, practice by Theme/LO, get adaptive feedback, and revisit misses in the Review Book; instructors set up a course, upload materials, and generate questions with the 3-agent AI pipeline.

Enrollment + course home Materials upload + RAG 3-agent generation Adaptive feedback + selection Review Book Mastery engine (L1 + L2)
⚑ Core loop live — one AI-generated, approved question served to a student end-to-end, with adaptive feedback and Review-Book capture.
Dev detail
WS-3/4Student — enrollment + roster cross-check, course home with coverage gating, question view, Strategy A/B/Adaptive feedback, selection ladder over Approved questions, Review Book (auto-collect + bookmark + re-practice), session summaries.
WS-5/6Instructor / AI — material ingestion, LLM classification, generator → structure-validator → reviewer pipeline (Draft output), course + Theme/LO CRUD, registration codes, review queue.
WS-7Mastery — Layer-1 rolling window + tier progression; Layer-2 LLM evaluator. Fallback: ship Layer-1-only if L2 isn’t stable.

Compressed from 2 weeks — the most aggressive week; the Layer-2 evaluator and stretch items are the first to slip.

The safety valve — how we protect the deadline

Full MVP by Sep 1 is aggressive for ~280 part-time hours, so phase order is the release valve: earlier phases are load-bearing, later ones are the first to slip. Phase 4’s test week is never traded for feature work.

  • 1stLayer-2 mastery evaluator → ship Layer-1-only progression first.
  • 2ndPhase 3 extras (analytics depth, TA/admin polish) move past Sep 1.
